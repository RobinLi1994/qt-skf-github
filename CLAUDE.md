# --- 核心使命与角色设定 ---
你是一个资深的 C++\QT 工程师，正在协助我开发一个名为 "wekey-skf" 的工具。
你的所有行动都必须严格遵守上面导入的项目宪法。

---
## 1. 技术栈与环境
- **语言**: C++
- **GUI 框架**: Qt 6.10.2
- **构建与运行**:
  - 使用 `Makefile` 进行标准化操作
  - 运行所有测试: `make test`
  - 构建并运行桌面程序: `make run`
- 优先使用ElaWidgetTools组件库中的功能，除非有明确的理由需要使用其他库。
---
## 2. Git与版本控制
- **Commit Message规范**: 严格遵循 Conventional Commits 规范。
  - 格式: `<type>(<scope>): <subject>`
  - 当被要求生成commit message时，必须遵循此格式。
  
    - `feat`: 新增功能
  
    - `fix`: 修复 Bug
  
    - `chore`: 代码优化，不涉及功能修改
  
    - `docs`: 文档更新
  
    - `style`: 代码格式化
  - **禁止**在 commit message 中添加 `Co-Authored-By` 行。

---
## 3. AI协作指令
- **当被要求添加新功能时**: 你的第一步应该是先用`@`指令阅读`internal/`下的相关包，并对照项目宪法，然后再提出你的计划。
- **当被要求编写测试时**: 你应该优先编写**表格驱动测试（Table-Driven Tests）**。
- **当被要求构建项目时**: 你应该优先提议使用`Makefile`中定义好的命令。

---
## 4. 日志规范
- **统一使用 Qt 日志宏**：`qDebug()`、`qInfo()`、`qWarning()`、`qCritical()`，禁止使用 `printf` / `std::cout`。
- **格式**：`[ClassName::methodName] 描述信息`，例如：`qDebug() << "[SkfPlugin::sign] container:" << containerName;`
- **级别规范**：
  - `qDebug()`：开发期调试信息，正常流程的关键步骤。
  - `qInfo()`：生产级别的重要事件。
  - `qWarning()`：可恢复的异常或非预期情况。
  - `qCritical()`：严重错误，操作无法继续。
- **禁止在日志中输出敏感信息**（PIN、私钥、密码等）。
